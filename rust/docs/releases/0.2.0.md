# Claw Code 0.2.0 release notes

`0.2.0` builds on the first release with session persistence, a substantially
reworked permission system, several UX fixes, and an opt-in built-in browser.

## Highlights

### Session persistence
- Conversations now survive a server restart. Each session is written to
  `~/.claw/sessions/<id>.json` (atomic write, `0600`) and restored on boot.
  Config, provider credentials, and MCP servers continue to round-trip through
  `~/.claw/state.json`. A single corrupt session file is skipped, not fatal.

### Permissions
- **Bash command classification.** `bash` is no longer a blanket
  `danger-full-access` tool. The actual command is parsed (quote-aware, split on
  `&&`/`||`/`;`/`|`) and judged by its riskiest segment, so `ls`/`find`/`grep`/
  `git status` run as read-only while `rm`/`curl`/`sudo`/`$(…)` still require
  full access. Output redirects to real files (`> out`, `2>err`, glued
  `x>file`) correctly require workspace-write.
- **Mid-turn mode switch.** Changing the permission mode now takes effect at the
  next tool boundary of an in-flight turn, not just the following turn.
- **Per-session "Allow always."** A new tray button remembers an approval for
  the rest of the session, keyed per command prefix (`bash:git`) or tool name.
  In-memory only — approvals reset on restart.

### UX
- **Pre-send validation** — a clear inline message when no model is selected or
  the model's provider has no API key, instead of a post-send backend error.
- **`/context`** command (alias of `/status`) showing context-window fill.
- **`/compact`** no longer leaves a stale system bubble that re-floats after
  every new message.
- **Response/thinking language** follows the user's language by default.
- **Live model lists** auto-fetch when a configured provider is expanded, and
  show `$in/$out` per-1M pricing on each model chip.
- **MCP registry** search results scroll correctly instead of overflowing.

### Built-in browser (opt-in)
- One-click **Browser** preset installs `@playwright/mcp`, giving the agent a
  real headed Chrome window with a dedicated persistent profile (logins survive
  restarts; you can take over in the window).
- Browser read tools (navigate/snapshot/screenshot) resolve to read-only;
  write tools (click/type/fill/evaluate/upload) require workspace-write.
- A **Browser** panel mirrors the agent's page (live screenshot + URL + DOM
  snapshot) while the agent is working, and leaves the window alone when idle.

## Notes
- The built-in browser requires Node.js 18+; first launch downloads Chromium.
- "Allow always" approvals and the browser profile are local, runtime-only state.
