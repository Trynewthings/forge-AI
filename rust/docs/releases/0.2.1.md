# Claw Code 0.2.1 release notes

A patch over 0.2.0 — browser-feature fixes, a "don't open the browser
unprompted" guardrail, and a per-session Usage panel.

## Browser
- **No surprise windows.** The Browser pane's live view now polls (and thus
  the screenshot tool that launches Chrome) **only while the agent is actively
  driving the browser** this turn. Unrelated questions, sessions without the
  browser attached, and idle periods never launch or resurrect a window.
- **Don't browse to read.** When the browser is available, the system prompt
  now tells the model to use `WebFetch` for reading/summarizing a URL and to
  reach for `browser_*` tools only when the user explicitly asks to operate a
  page (log in, click, type, fill, multi-step) — never opening a window
  unprompted. (The agent reads accessibility-tree text, not screenshots, so
  this loses nothing for non-multimodal models.)
- **Tighter permissions.** `browser_evaluate` (arbitrary JS on a possibly
  logged-in page) and `browser_install` (system-level browser download) now
  require `danger-full-access`, matching `browser_run_code_unsafe` /
  `browser_network_request`. Plain page interaction stays `workspace-write`.
- Fixed a live-view edge where a stale URL from a previous turn could be
  misread as a user-close and stop mirroring.

## Usage
- New **Settings → Usage** tab (per active session): model, turns, context
  fill vs. budget, total input/output (and cache-read) tokens, last-turn
  breakdown, and a rough USD cost estimate from the local price table. Backed
  by a new `GET /sessions/{id}/usage` endpoint.

## Notes
- The cost figure is a rough estimate (cache discounts aren't modeled).
- The built-in browser still requires Node.js 18+; first launch downloads Chromium.
